# Requirements Document — AI Daily Notes (MVP)

## 1. User Personas

### Персона 1: Алексей, 28 лет
- **Род занятий:** IT-специалист
- **Цель:** Ведёт личный дневник для рефлексии, хочет быстро записывать мысли и возвращаться к ним позже
- **Боли:** Заметки разбросаны по разным приложениям, нет единого места для личных записей
- **Ожидания:** Простой интерфейс, быстрый поиск, теги для организации

### Персона 2: Мария, 35 лет
- **Род занятий:** Писательница
- **Цель:** Использует ежедневник для идей, черновиков и наблюдений
- **Боли:** Тяжело вспоминать, какие темы поднимала за неделю; нужен экспорт для внешних инструментов
- **Ожидания:** Минималистичный редактор, еженедельная сводка, экспорт в Markdown

---

## 2. User Stories

### 2.1 Регистрация
**Как** пользователь, **я хочу** зарегистрироваться в приложении, **чтобы** получить доступ к личному ежедневнику.

#### Критерии приёмки (BDD)
- **Дано:** я новый пользователь с email и паролем
- **Когда:** я отправляю форму регистрации с корректными данными
- **Тогда:** создаётся мой аккаунт, и я автоматически вхожу в приложение

- **Дано:** я новый пользователь
- **Когда:** я отправляю форму регистрации с email, который уже занят
- **Тогда:** я вижу сообщение «Этот email уже зарегистрирован»

---

### 2.2 Вход в систему
**Как** зарегистрированный пользователь, **я хочу** войти в приложение, **чтобы** просматривать свои заметки.

#### Критерии приёмки (BDD)
- **Дано:** я зарегистрирован с email и паролем
- **Когда:** я ввожу корректный email и пароль
- **Тогда:** я попадаю на главный экран с моими заметками

- **Дано:** я зарегистрированный пользователь
- **Когда:** я ввожу неверный пароль
- **Тогда:** я вижу сообщение «Неверный email или пароль»

---

### 2.3 Создание заметки
**Как** пользователь, **я хочу** создать заметку с заголовком и содержимым, **чтобы** записать свою мысль.

#### Критерии приёмки (BDD)
- **Дано:** я нахожусь на экране создания заметки
- **Когда:** я ввожу заголовок и текст и нажимаю «Сохранить»
- **Тогда:** заметка сохраняется с текущей датой и отображается в списке заметок за этот день

- **Дано:** я создаю заметку
- **Когда:** я оставляю заголовок пустым
- **Тогда:** заметка сохраняется с заголовком «Без названия»

---

### 2.4 Просмотр заметок по датам
**Как** пользователь, **я хочу** видеть заметки за выбранную дату, **чтобы** просматривать записи по дням.

#### Критерии приёмки (BDD)
- **Дано:** у меня есть заметки за разные даты
- **Когда:** я выбираю конкретную дату в календаре
- **Тогда:** отображаются только заметки за выбранную дату

- **Дано:** за выбранную дату нет заметок
- **Когда:** я открываю эту дату
- **Тогда:** я вижу сообщение «Нет заметок на этот день» и кнопку «Создать заметку»

---

### 2.5 Редактирование заметки
**Как** пользователь, **я хочу** отредактировать существующую заметку, **чтобы** исправить или дополнить её.

#### Критерии приёмки (BDD)
- **Дано:** я открыл существующую заметку
- **Когда:** я изменяю заголовок или текст и нажимаю «Сохранить»
- **Тогда:** изменения сохраняются, и я вижу обновлённую заметку

- **Дано:** я редактирую заметку
- **Когда:** я нажимаю «Отмена»
- **Тогда:** никакие изменения не сохраняются, я возвращаюсь к просмотру заметки

---

### 2.6 Удаление заметки
**Как** пользователь, **я хочу** удалить заметку, **чтобы** избавиться от ненужных записей.

#### Критерии приёмки (BDD)
- **Дано:** я просматриваю существующую заметку
- **Когда:** я нажимаю «Удалить» и подтверждаю действие
- **Тогда:** заметка удаляется без возможности восстановления

- **Дано:** я нажимаю «Удалить»
- **Когда:** появляется диалог подтверждения и я нажимаю «Отмена»
- **Тогда:** заметка остаётся без изменений

---

### 2.7 Поиск по тексту
**Как** пользователь, **я хочу** искать заметки по тексту, **чтобы** быстро находить нужную информацию.

#### Критерии приёмки (BDD)
- **Дано:** у меня есть несколько заметок с разным текстом
- **Когда:** я ввожу поисковый запрос
- **Тогда:** отображаются все заметки, содержащие этот текст в заголовке или содержимом

- **Дано:** я ввёл поисковый запрос
- **Когда:** ни одна заметка не соответствует запросу
- **Тогда:** я вижу сообщение «Ничего не найдено»

---

### 2.8 Добавление тегов к заметке
**Как** пользователь, **я хочу** добавлять теги к заметке, **чтобы** категоризировать записи по темам.

#### Критерии приёмки (BDD)
- **Дано:** я создаю или редактирую заметку
- **Когда:** я ввожу название тега и добавляю его к заметке
- **Тогда:** тег сохраняется и отображается на карточке заметки

- **Дано:** я добавляю тег к заметке
- **Когда:** я ввожу тег, который уже существует в моей коллекции
- **Тогда:** используется существующий тег (дубликаты не создаются)

---

### 2.9 Фильтрация заметок по тегу
**Как** пользователь, **я хочу** фильтровать заметки по тегу, **чтобы** видеть записи по определённой теме.

#### Критерии приёмки (BDD)
- **Дано:** у меня есть заметки с разными тегами
- **Когда:** я выбираю тег из списка
- **Тогда:** отображаются только заметки с выбранным тегом

- **Дано:** я просматриваю отфильтрованные заметки
- **Когда:** я снимаю фильтр по тегу
- **Тогда:** отображаются все мои заметки

---

### 2.10 Экспорт заметки в Markdown
**Как** пользователь, **я хочу** экспортировать заметку в Markdown, **чтобы** использовать её в других инструментах.

#### Критерии приёмки (BDD)
- **Дано:** я просматриваю заметку
- **Когда:** я нажимаю «Экспорт в Markdown»
- **Тогда:** файл .md скачивается на моё устройство с содержимым заметки

- **Дано:** я экспортирую заметку
- **Когда:** файл создаётся
- **Тогда:** имя файла содержит дату заметки и заголовок

---

### 2.11 Еженедельная AI-суммаризация
**Как** пользователь, **я хочу** получать AI-суммаризацию моих заметок за неделю, **чтобы** видеть ключевые темы и инсайты.

#### Критерии приёмки (BDD)
- **Дано:** у меня есть заметки за последние 7 дней
- **Когда:** я нажимаю «Суммаризация за неделю»
- **Тогда:** генерируется краткое резюме на русском языке (3-5 предложений) с ключевыми темами недели

- **Дано:** за последние 7 дней нет заметок
- **Когда:** я нажимаю «Суммаризация за неделю»
- **Тогда:** я вижу сообщение «Недостаточно заметок для суммаризации»

- **Дано:** я запросил суммаризацию
- **Когда:** генерация завершена
- **Тогда:** результат отображается в модальном окне с возможностью закрыть или скопировать

---

### 2.12 Тёмная тема
**Как** пользователь, **я хочу** переключать тему оформления, **чтобы** вести записи с комфортом в любое время суток.

#### Критерии приёмки (BDD)
- **Дано:** я нахожусь в приложении
- **Когда:** я нажимаю переключатель темы
- **Тогда:** интерфейс переключается между светлой и тёмной темой

- **Дано:** я выбрал тёмную тему
- **Когда:** я перезагружаю страницу
- **Тогда:** тёмная тема сохраняется (выбор хранится в localStorage или в профиле)

---

## 3. Domain Model

### 3.1 Сущности

```
┌─────────────────────────┐
│         User            │
├─────────────────────────┤
│ id (UUID, PK)           │
│ email (unique)          │
│ password_hash           │
│ name                    │
│ theme_preference        │
│ created_at              │
│ updated_at              │
└──────────┬──────────────┘
           │ 1
           │
           │ has
           │
     ┌─────┴──────────────┐
     │        Note         │
     ├─────────────────────┤
     │ id (UUID, PK)       │
     │ user_id (FK)        │
     │ title               │
     │ content             │
     │ date                │
     │ created_at          │
     │ updated_at          │
     └──────────┬──────────┘
                │ M
                │
                │ has
                │
     ┌──────────┴──────────┐
     │   Note_Tag (M:M)    │
     ├─────────────────────┤
     │ note_id (FK)        │
     │ tag_id (FK)         │
     └──────────┬──────────┘
                │ M
                │
     ┌──────────┴──────────┐
     │        Tag          │
     ├─────────────────────┤
     │ id (UUID, PK)       │
     │ user_id (FK)        │
     │ name                │
     │ created_at          │
     └─────────────────────┘
```

### 3.2 Связи

| Связь | Тип | Описание |
|-------|-----|----------|
| User → Note | One-to-Many | Один пользователь может иметь много заметок |
| Note → Tag | Many-to-Many | Заметка может иметь много тегов, тег может быть на многих заметках |
| User → Tag | One-to-Many | Один пользователь может создать много тегов (область видимости тегов) |

### 3.3 Описание полей

#### User
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный идентификатор |
| email | String | Email для входа |
| password_hash | String | Хеш пароля (bcrypt) |
| name | String | Отображаемое имя |
| theme_preference | Enum('light', 'dark') | Выбранная тема |
| created_at | Timestamp | Дата регистрации |
| updated_at | Timestamp | Дата обновления профиля |

#### Note
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный идентификатор |
| user_id | UUID (FK → User) | Владелец заметки |
| title | String | Заголовок (может быть пустым) |
| content | Text | Содержимое заметки |
| date | Date | Дата заметки (выбранная пользователем) |
| created_at | Timestamp | Дата создания |
| updated_at | Timestamp | Дата последнего редактирования |

#### Tag
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный идентификатор |
| user_id | UUID (FK → User) | Владелец тега |
| name | String | Название тега |
| created_at | Timestamp | Дата создания |

## 4. Non-functional Requirements

| # | Требование | Описание | Критерий приёмки |
|---|------------|----------|------------------|
| NFR1 | Производительность | Время загрузки основной страницы | ≤ 2 секунды при стабильном интернете |
| NFR2 | Доступность | Приложение доступно 24/7 | ≥ 99.9% uptime (хостинг Vercel) |
| NFR3 | Безопасность | Пароли хранятся в захешированном виде | Использовать bcrypt, не хранить plain-text |
| NFR4 | Безопасность | Все данные передаются по HTTPS | Включено по умолчанию на Vercel |
| NFR5 | Удобство использования | Минималистичный интерфейс без лишних элементов | Навигация интуитивна, не требует обучения |
| NFR6 | Кроссплатформенность | Базовая адаптивность под мобильные устройства | Интерфейс читаем на экранах от 375px |
| NFR7 | Надёжность | Автосохранение черновика | Текст не теряется при случайном закрытии вкладки |
| NFR8 | Совместимость | Поддержка современных браузеров | Chrome, Firefox, Safari, Edge (2 последние версии) |

## 5. MVP Scope Summary

### Входит в MVP
- [x] Регистрация и вход (email + пароль)
- [x] CRUD заметок (создание, чтение по дате, редактирование, удаление)
- [x] Поиск по тексту заметок
- [x] Ручное добавление тегов
- [x] Фильтрация заметок по тегу
- [x] Экспорт одной заметки в Markdown
- [x] AI-суммаризация заметок за неделю
- [x] Тёмная тема
- [x] Базовая мобильная адаптивность

### НЕ входит в MVP (будущие версии)
- [ ] Совместный доступ к заметкам
- [ ] Категории / папки
- [ ] Медиа-вложения (изображения, файлы)
- [ ] Push-уведомления
- [ ] Полноценное мобильное приложение
- [ ] Авто-тегирование AI
- [ ] Экспорт всех заметок архивом
- [ ] Версионирование заметок (history)
